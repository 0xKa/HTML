Scalable Vector Graphics, <svg> element

Canvas vs SVG: Canvas and SVG are both used for drawing graphics on the web, but they work in different ways:

  • Canvas: Uses a pixel-based, immediate drawing method. Once drawn, graphics are part of a bitmap, which is efficient for dynamic, complex, or frequently changing visuals but makes individual element manipulation harder.
  • SVG: Uses a vector-based, retained drawing method. Graphics are defined as XML elements, allowing them to be scalable, easily styled with CSS, and individually manipulated with JavaScript, making them ideal for static or interactive visuals with crisp edges.

Core Concepts:

  • What is SVG?
    • SVG stands for Scalable Vector Graphics.
    • It allows for the creation and manipulation of vector graphics such as shapes, paths, and text.
    • Because it's vector-based, SVG images can scale to fit different screen sizes and resolutions without losing quality.
  • Benefits of SVG
    • Scalability and Resolution Independence: Ideal for responsive design settings.
    • Interactivity and Animation: SVG graphics can be animated and made interactive using CSS and JavaScript.
    • Accessibility: Text labels and descriptions can be added, making SVGs accessible to screen readers.
    • Performance: Usually smaller in file size compared to other image formats, especially when used for complex graphics.
  • Basic SVG Syntax
    • SVG graphics are defined in XML, the same syntax used in XHTML.
    • You can embed SVG directly into HTML documents.
