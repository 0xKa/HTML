<form> Tag and Inputs - Dark Theme Example

The <form> element allows user input submission. Below, inputs are grouped using <fieldset> and labeled with <legend>.

Key attributes that configure the form element's functionality include:

  • action: Specifies the URL of the server script that handles the form data after submission.
  • method: Defines the HTTP method used to send the form data (GET or POST).
  • enctype: Specifies how the form data should be encoded when sending to the server. Common values include application/x-www-form-urlencoded, multipart/form-data (for file uploads), and text/plain.

method attribute: GET vs POST:

GET:

Description: Data is appended to the URL as a query string and visible to anyone. It has a length limitation and should not be used for sensitive information.

Use Cases: Suitable for form submissions where the data does not include sensitive information, such as search filters or query parameters.

POST:

Description: Data is sent through the HTTP header, so the information is not displayed in the URL. This method allows sending large quantities of data and is secure for sensitive information.

Use Cases: Ideal for updating data, sending sensitive information (like passwords), and for any form that results in a change of state or data manipulation on the server.
